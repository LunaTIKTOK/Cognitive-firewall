# Cognitive Firewall Aperture Model

The aperture is the governed opening between agent reasoning and real-world execution. It determines how much action, if any, an agent is allowed to take.

- `ALLOW` = open aperture: execution may proceed through governance token path.
- `SPECULATE` = narrowed aperture: execution may proceed only with explicit caps and falsification conditions.
- `BLOCK` = closed aperture: no token issuance, no execution.

Token issuance occurs only after aperture checks pass (domain mismatch, uncertainty, simulation, runtime governance) so unqualified intent cannot mint execution authority.

Unsupported, incoherent, or contradictory intent closes the aperture because execution authority must be bounded by explicit and auditable assumptions.
